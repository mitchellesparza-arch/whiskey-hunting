The headline

Six moves that do most of the work.

If the sprint stops at six things, do these. They drive 80% of the perceived lift.

01

Real icons in the bottom nav

Swap emoji for line icons (lucide). Active state becomes a copper gradient pill above the icon, not a colored emoji. This is the single highest-impact change.

02

Lift to a real component library

Eight UI primitives in app/components/ui/. No more inline-style sprawl. <Card>, <Chip>, <Button>, <StatTile> become the new default vocabulary.

03

Tighten the palette to copper

The current orange (#e8943a) reads playful. Shift the brand 8° warmer to a copper #d97e2c, introduce a copper gradient for hero moments, and de-saturate functional colors so they sit calmly on bourbon dark.

04

Editorial typography

Add Fraunces (italic) for hero numbers, scores, and section openers. DM Sans stays for UI. The blind-tasting result screen and the auction discount badges become moments of typographic delight.

05

Motion as feedback

Press states (scale(.97)), spring sheets, the slot-machine reveal in Pick My Pour, a subtle pulse on auctions ending in < 2h. Nothing flashy — just a system that responds.

06

Empty states that feel like the club

Today's empty states are placeholder text. Replace with hero-treatment empty states using the copper gradient, a hunting-themed line, and a single CTA. This is the front door for new members.

"Make it feel like a Chicago speakeasy that happens to be a PWA — not a CRUD app with a brown skin."

What we found

The product works. The system has drift.

01

The bourbon-on-dark direction

The warm dark palette is genuinely on-brief and rare in the App Store. Don't pivot to light mode. Refine the ramp, deepen the base, push the accent warmer. The bones are right.

02

DM Sans + DM Mono

Solid pairing. Neutral, modern, monospace works for UPCs and timestamps. We'll keep both and add Fraunces italic for headlines and hero numbers — the "editorial" voice the app is missing.

03

The information architecture

Five tabs is right. Finds-as-home is right. The Profile cluster (Collection, Wishlist, Friends, Pour, Blind) is genuinely well-designed. We're not touching IA.

04

Tokens exist, but aren't used

Twelve CSS variables are defined in globals.css. Every screen still ships with hardcoded hex values — #1a1008, #9a7c55, #e8943a appear ~140× across app/. This is the work.

05

No shared component layer

The "card" pattern is reimplemented inline on every page with slightly different padding, border, hover, radius. Same with chips, badges, stat tiles, sheets. There's a .card Tailwind class that's used inconsistently.

06

Type scale drifts

I count 9 distinct font-sizes across the codebase (10, 11, 11.5, 12, 13, 14, 15, 16, 22). Lock to 7. The body/meta/overline rhythm is what makes a UI feel "considered."

07

Emoji as UI furniture

🥃📍🔔⚙️🚛🦄🎲🙈 do double-duty as both nav icons and content. The result reads like a 2018 hackathon project. Move emoji to content only — replace nav, button, and chip emoji with lucide line icons.

08

Empty states are placeholder text

Most "no results" states are a centered gray sentence. The Tracker empty state is the exception (already strong) — make every other empty state match its level.

09

No motion language

The app has zero hover/press feedback. No spring on the drawers. No animation on Pick My Pour reveal. This is the easiest win for "feels expensive."

What "good" looks like

Five principles, five references.

Every redesign decision in this doc traces back to one of these. The references aren't templates — they're the *quality bar* we're targeting in a bourbon-club idiom.

Reference · Things 3

Quiet, generous, considered.

The screen tells you what's important by what it omits, not by what it shouts. We adopt this in stat tiles, list rows, and section headers — fewer borders, more whitespace, single-source emphasis.

Reference · Linear

System over decoration.

One way to make a card. One way to make a chip. One way to make a button. Variants live in props, not in re-implementations. The result is a codebase you can refactor without breaking the design.

Identity through restraint.

Strava picks one accent (orange) and protects it. We do the same with copper. Functional colors stay desaturated so the brand color always wins the visual hierarchy fight.

Content on top, chrome out of the way.

Kickoff prompt for Claude Code

Copy and paste verbatim. The full version lives in redesign/PROMPT.md — this is the abbreviated kickoff.

You're picking up Tater Tracker, a private-club bourbon PWA built on Next.js 14. The product works — the design system has drifted. I've prepared a refresh package at /redesign. Read it first: 1. redesign/Design Check-In.html — the audit, principles, and before/after mockups. Open this in your browser; it's the visual source of truth for the sprint. 2. redesign/PROMPT.md — full working principles, scope, and done criteria. Read once, then refer back. 3. redesign/SPRINT.md — the day-by-day checklist. Work it top to bottom. 4. redesign/tokens.css — the new token set. Replace the :root block in app/globals.css with this; legacy aliases are preserved so existing code keeps compiling. Scope: pixel-only. No backend changes, no schema changes, no new features. The IA, the routes, the Redis keys, the ELO math, the Discord webhooks — all stay untouched. Working principles (non-negotiable): • Every color, radius, spacing, font-size in new code is a var(--…) from tokens.css. No hex literals in style props. • Lift any pattern that appears on >1 screen into app/components/ui/. Eight primitives are specified in the design doc — build those first. • Replace emoji used as icons (nav, buttons, chips, headers, empty states) with lucide-react line icons. Emoji stays for content (member-typed notes, the 🥃 brand mark). • Use the locked 7-size type scale. Don't introduce new sizes. • Every interactive element gets a press state and a transition. Start with Day 1 in SPRINT.md. When you complete a day, write a short status comment in PR description — what was touched, what was deferred, what surprised you. Ask before deviating from the design doc. The bourbon-on-dark direction is the right one. We're refining it, not pivoting. Make it feel like a Chicago speakeasy that happens to be a PWA — not a CRUD app with a brown skin.
